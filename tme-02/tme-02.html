Étape 2: les classifieurs¶

Un classifieur (binaire dans ce cours) permet, pour chaque point d'entre donn, de calculer un score (qui est une valeur de $\mathbb{R}$) pour dterminer l'tiquette d'un exemple: si le score est strictement ngatif, la donne est associe l'tiquette $-1$, sinon elle est associe l'tiquette $+1$.

Représentation orientée objets¶

Pour implmenter nos algorithmes d'apprentissage et pour reprsenter les donnes, nous allons utiliser quelques lments de programmation oriente objet (POO) qu'autorise le langage Python.

Si vous ne connaissez rien la POO, il est fortement conseill de vous familiariser avec elle, par exemple en commenant par lire une documentation ou suivre un tutorial sur le web (par exemple, http://apprendre-python.com/page-apprendre-programmation-orientee-objet-poo-classes-python-cours-debutants).

La classe Classifier¶

Les diffrents classifieurs que l'on va implmenter seront drivs d'une classe unique : la classe `Classifier` qui est donne ci-dessous. Pour nous, cette classe est l'quivalent une classe "abstraite" de Java, elle devra tre drive par hritage pour dfinir des classifieurs particuliers qui seront vus ce semestre (knn, perceptron, arbres de dcision, etc.). Cette classe `Classifier` contient 5 mthodes: - `__init__`: qui permet d'initialiser les paramtres du classifieur et qui sera donc propre chaque classifieur. Les classes filles tendront donc cette mthode en rajoutant les paramtres qui les concernent. Le paramtre obligatoire donner la cration d'un classifieur est le nombre de dimensions des donnes qu'il aura traiter. - `train`: c'est la mthode qui permet d'entraner le modle, on doit donc donner en argument le dataset qui sert l'entranement. - `score`: cette mthode est utilise une fois le classifieur entrain. Elle prend en argument la description d'un exemple et elle rend un score (une valeur de $\mathbb{R}$). - `predict`: cette mthode, comme la prcdente, s'utilise une fois le classifieur entran. Elle rend $\hat y$, le label (-1 ou +1) qui est prdit par le classifieur pour la description de l'exemple donne en argument. Gnralement, cette valeur de prdiction $\hat y$ s'obtient partir du score: si le score est strictement ngatif $\hat y$ vaut $-1$, sinon il vaut $+1$. - `accuracy`: comme les 2 prcdentes, cette mthode s'utilise une fois le classifieur entran. Elle permet de calculer la qualit du classifieur sur un dataset donn, cette qualit est mesure par le taux de bonne classification du classifieur sur le dataset. C'est donc une valeur de $[0,1]$ qui s'obtient divisant le nombre d'exemples du dataset qui sont bien classs par le classifieur par le nombre total d'exemples du dataset. Les mthodes `train`, `score` et `predict` ne seront pas dfinies dans la classe `Classifier` mais dans les classes qui hriteront ce cette classe car ces mthodes dpendent du modle que l'on choisira d'implmenter. Par contre, la fonction `accuracy` peut tre dfinie ds maintenant dans cette classe `Classifier`.

les $k$-plus proches voisins ($k$-ppv ou $k$-NN)¶

Le classifieur des $k$-plus proches voisins (en anglais: $k$-nearest neighbors) est trs intuitif: il s'agit de prdire la classe majoritaire dans le voisinage d'un point en examinant ses $k$ plus proches voisins. La figure ci-dessous illustre bien le principe gnral de l'algorithme: Cet algorithme d'apprentissage est particulier: il n'y a pas d'apprentissage, il faut juste stocker tous les exemples d'apprentissage pour les rutiliser plus tard !! En contrepartie, l'infrence coute trs cher: pour chaque prdiction , il faut analyser tout le dataset stock afin de trouver les $k$ plus proches voisins de l'exemple prdire.
Description de la classe `ClassifierKNN` (qui tend `Classifier`): - dans le constructeur, on prcise, en plus de la dimension du dataset, le nombre de voisins utiliser (c'est la valeur $k$) - dans la mthode `train`, on donne le dataset qui doit servir de dataset de rfrence et qui va donc tre stock (la particularit de l'algorithme des $k$-plus proches voisins) - dans la mthode `score`, pour une description d'exemple $x$ donne : - 1) on construit le tableau des distances entre $x$ et les exemples du dataset de rfrence, - 2) on trie ce tableau grce, par exemple, la mthode `numpy.argsort`, - 3) le score qui doit tre rendu est calcul partir de **la proportion d'exemples de classe +1** parmi les $k$ plus proches voisins de $x$. Le score - vaut $0$ si la proportion est d'exactement $50\%$ d'exemples de classe +1 parmi les $k$ plus proches voisins; - est strictement infrieur $0$ si la proportion est strictement infrieure $50\%$. Il est d'autant plus ngatif que la proportion est faible; - est strictement suprieur $0$ si la proportion est strictement suprieur $50\%$. Il est d'autant plus grand que la proportion est importante.
Par exemple, le score peut se calculer partir de $p \in [0,1]$, la proportion d'exemples, par $2(p-0.5)$. - dans la mthode `predict`, pour une description d'exemple $x$ donne, on utilise le score rendu par la fonction prcdente pour rendre la classe de l'exemple, c'est--dire soit $-1$ soit $+1$.

def plot_frontiere(desc_set, label_set, classifier, step=30):
    """ desc_set * label_set * Classifier * int -> NoneType
        Remarque: le 4e argument est optionnel et donne la "rsolution" du trac: plus il est important
        et plus le trac de la frontire sera prcis.        
        Cette fonction affiche la frontire de dcision associe au classifieur
    """
    mmax=desc_set.max(0)
    mmin=desc_set.min(0)
    x1grid,x2grid=np.meshgrid(np.linspace(mmin[0],mmax[0],step),np.linspace(mmin[1],mmax[1],step))
    grid=np.hstack((x1grid.reshape(x1grid.size,1),x2grid.reshape(x2grid.size,1)))
    
    # calcul de la prediction pour chaque point de la grille
    res=np.array([classifier.predict(grid[i,:]) for i in range(len(grid)) ])
    res=res.reshape(x1grid.shape)
    # tracer des frontieres
    #colors[0] est la couleur des -1 et colors[1] est la couleur des +1
    plt.contourf(x1grid,x2grid,res,colors=["darksalmon","skyblue"],levels=[-1000,0,1000])

Un classifieur linéaire¶

Nous allons maintenant dfinir un classifieur linaire alatoire en tendant la classe Classifier et en implmentant le constructeur ainsi que les 3 mthodes predict, score et train.

Ce classifieur est un des plus simple que l'on puisse construire: on gnre alatoirement une droite (ou hyperplan si on a plus de 2 dimensions) dans l'espace des dimensions en tirant alatoirement un vecteur $w$ donnant les coefficients de chaque dimensions. L'quation de la droite (hyperplan) est alors donne par $\langle x, w\rangle = 0$ (cf. TD de la semaine 1).

Cela nous permis de construire un vecteur de poids. Cette droite est utilise pour classer les exemples: d'un ct de la droite ils sont prdits de la classe +1, de l'autre ct ils sont prdits de la classe -1.

Naturellement, ce classifieur a de bonnes chances de ne pas tre trs efficace...

Pour un classifieur linaire alatoire: - dans le constructeur, on doit gnrer alatoirement un vecteur $w$ de taille `input_dimension` de valeurs dans $[-1,1]$. Les poids doivent ensuite normaliss (c'est--dire que la norme de $w$ doit tre gale 1). Ainsi, si $v$ est le vecteur alatoire valeurs dans $[-1,1]$ gnr, on obtient un vecteur $w$ normalis partir de $v$ en prenant chaque composante de $v$ est en la divisant par la norme de $v$. - pour ce classifieur, la mthode `train` ne fait rien. On ne mettra qu'un print permettant d'afficher le message "Pas d'apprentissage pour ce classifieur" - pour une description d'exemple $x$, la mthode `score` rend la valeur de $\langle x, w\rangle$ - pour une description d'exemple $x$, la mthode `predict` doit utiliser la valeur de $\langle x, w\rangle$ pour prdire le label (+1 ou -1) associ
