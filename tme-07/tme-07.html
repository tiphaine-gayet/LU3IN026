Entropie et gain d'information

On note $\{c_1, c_2,..., c_k\}$, l'ensemble des valeurs de classes possibles dans un ensemble $Y$.

On note $p_1$ la probabilit de la classe $c_1$ parmi $Y$, etc., et on note $P=(p_1, p_2,..., p_k)$ la distribution de probabilits sur les classes (on a $\sum_{i=1}^{k} p_i =1$ et $p_i \in [0,1]$ pour tout $i$).

L'entropie de Shannon de la distribution $P$ est donne par:

$ H_S(P) = -\sum_{i=1}^{k} p_i \log_k(p_i)$

Le logarithme utilis est le logarithme en base $k$ afin d'obtenir une valeur comprise entre $0$ et $1$ de l'entropie.

Pour ce qui suit, une distribution de probabilits est reprsente par une liste Python $P=[p_1,...,p_k]$ de valeurs relles telle que

  • pour tout $i=1,..., k$ : $0 \leq p_i \leq 1$
  • $\sum_{i=1}^{k}p_i = 1$

Reprsentation d'un arbre en Python

Pour reprsenter un arbre en Python, on a besoin de dfinir une structure de donnes adquate.

Un arbre de dcision est dfini par des noeuds qui sont de 2 types :

  • noeud interne : c'est un noeud qui est associ un attribut (ie. une feature de description des exemples) et qui possde des fils qui sont aussi des noeuds.
  • feuille : c'est un noeud qui est associ un label de la classe et qui a la particularit de ne pas avoir de descendants.

Pour reprsenter des noeuds, nous dfinissons la classe NoeudCategoriel suivante:

class NoeudCategoriel:
    """ Classe pour reprsenter des noeuds d'un arbre de dcision
    """
    def __init__(self, num_att=-1, nom=''):
        """ Constructeur: il prend en argument
            - num_att (int) : le numro de l'attribut auquel il se rapporte: de 0  ...
              si le noeud se rapporte  la classe, le numro est -1, on n'a pas besoin
              de le prciser
            - nom (str) : une chane de caractres donnant le nom de l'attribut si
              il est connu (sinon, on ne met rien et le nom sera donn de faon 
              gnrique: "att_Numro")
        """
        self.attribut = num_att    # numro de l'attribut
        if (nom == ''):            # son nom si connu
            self.nom_attribut = 'att_'+str(num_att)
        else:
            self.nom_attribut = nom 
        self.Les_fils = None       # aucun fils  la cration, ils seront ajouts
        self.classe   = None       # valeur de la classe si c'est une feuille
        
    def est_feuille(self):
        """ rend True si l'arbre est une feuille 
            c'est une feuille s'il n'a aucun fils
        """
        return self.Les_fils == None
    
    def ajoute_fils(self, valeur, Fils):
        """ valeur : valeur de l'attribut de ce noeud qui doit tre associe  Fils
                     le type de cette valeur dpend de la base
            Fils (NoeudCategoriel) : un nouveau fils pour ce noeud
            Les fils sont stocks sous la forme d'un dictionnaire:
            Dictionnaire {valeur_attribut : NoeudCategoriel}
        """
        if self.Les_fils == None:
            self.Les_fils = dict()
        self.Les_fils[valeur] = Fils
        # Rem: attention, on ne fait aucun contrle, la nouvelle association peut
        # craser une association existante.
    
    def ajoute_feuille(self,classe):
        """ classe: valeur de la classe
            Ce noeud devient un noeud feuille
        """
        self.classe    = classe
        self.Les_fils  = None   # normalement, pas obligatoire ici, c'est pour tre sr
        
    def classifie(self, exemple):
        """ exemple : numpy.array
            rend la classe de l'exemple 
            on rend la valeur None si l'exemple ne peut pas tre class (cf. les questions
            poses en fin de ce notebook)
        """
        if self.est_feuille():
            return self.classe
        if exemple[self.attribut] in self.Les_fils:
            # descente rcursive dans le noeud associ  la valeur de l'attribut
            # pour cet exemple:
            return self.Les_fils[exemple[self.attribut]].classifie(exemple)
        else:
            # Cas particulier : on ne trouve pas la valeur de l'exemple dans la liste des
            # fils du noeud... Voir la fin de ce notebook pour essayer de rsoudre ce mystre...
            print('\t*** Warning: attribut ',self.nom_attribut,' -> Valeur inconnue: ',exemple[self.attribut])
            return None
    
    def compte_feuilles(self):
        """ rend le nombre de feuilles sous ce noeud
        """
        raise NotImplementedError("A implmenter plus tard (voir plus loin)")
     
    def to_graph(self, g, prefixe='A'):
        """ construit une reprsentation de l'arbre pour pouvoir l'afficher graphiquement
            Cette fonction ne nous intressera pas plus que a, elle ne sera donc pas explique            
        """
        if self.est_feuille():
            g.node(prefixe,str(self.classe),shape='box')
        else:
            g.node(prefixe, self.nom_attribut)

                g.edge(prefixe,prefixe+str(i), valeur)
                i = i+1        
        return g

Construction de l'arbre de dcision

Comme on l'a vu dans le cours, pour construire un arbre de dcision partir d'un dataset ($X$, $Y$), il faut raliser les tapes suivantes:

  1. calculer l'entropie de Shannon de l'ensemble des classes $Y$, on la note $H_S(Y)$.

  2. si $H_S(Y)$ est infrieure epsilon (qui est un rel positif donn en paramtre de l'algorithme), alors construire une feuille avec cet ensemble, la classe associe cette feuille est la classe majoritaire dans $Y$.

  3. sinon, pour chaque attribut $X_j$ qui dcrit les exemples de $X$,

    3.1. pour chacune des valeurs $v_{jl}$ de $X_j$ construire l'ensemble des exemples de $X$ qui possde la valeur $v_{jl}$ ainsi que l'ensemble de leurs labels.

    3.2. calculer l'entropie conditionnelle de Shannon de la classe relativement l'attribut $X_j$. On note $H_S(Y|X_j)$ cette entropie

  4. l'attribut $X_{best}$ qui maximise le gain d'information est choisi pour constituer un nouveau noeud $\eta$ de l'arbre de dcision:

    4.1. chaque valeur de $X_{best}$ est utilise pour dcomposer ($X$, $Y$) en autant de datasets que $X_{best}$ possde de valeurs: chaque dataset est sparant $X$ et $Y$ selon la valeur prise pour $X_{best}$.

    4.2. pour chaque dataset obtenu on reprend en 1 pour construire chacun des fils de $\eta$.

On rappelle que le gain d'information est:

$$I_S(X_j,Y) = H_S(Y) - H_S(Y|X_j)$$

Remarque: si le gain d'information est nul pour tous les attributs, alors on arrte la dcomposition et on construit une feuille avec cet ensemble en donnant la classe majoritaire comme tiquette cette feuille.

        epsilon : seuil d'entropie pour le critre d'arrt 
        LNoms : liste des noms de features (colonnes) de description

    entropie_ens = entropie(Y)
    if (entropie_ens <= epsilon):
        # ARRET : on cre une feuille
        noeud = NoeudCategoriel(-1,"Label")
        noeud.ajoute_feuille(classe_majoritaire(Y))
    else:
        min_entropie = 1.1
        i_best = -1
        Xbest_valeurs = None
        
        #############
        
        # COMPLETER CETTE PARTIE : ELLE DOIT PERMETTRE D'OBTENIR DANS
        # i_best : le numro de l'attribut qui minimise l'entropie
        # min_entropie : la valeur de l'entropie minimale
        # Xbest_valeurs : la liste des valeurs que peut prendre l'attribut i_best
        #
        # Il est donc ncessaire ici de parcourir tous les attributs et de calculer
        # la valeur de l'entropie de la classe pour chaque attribut.
        
        
        #############################################

        if (entropie_ens - min_entropie) == 0: # pas de gain d'information possible
            # ARRET : on cre une feuille
            noeud = NoeudCategoriel(-1,"Label")
            noeud.ajoute_feuille(classe_majoritaire(Y))
            
        if len(LNoms)>0:  # si on a des noms de features
            noeud = NoeudCategoriel(i_best,LNoms[i_best])    
        else:
            noeud = NoeudCategoriel(i_best)
        for v in Xbest_valeurs:
            noeud.ajoute_fils(v,construit_AD(X[X[:,i_best]==v], Y[X[:,i_best]==v],epsilon,LNoms))
    return noeud

Evaluation "Leave one out"

Nous avons vu 2 approches pour valuer un algorithme d'apprentissage:

  • la mesure de l'accuracy sur les donnes d'apprentissage et sur une base de test
  • la ralisation d'une validation croise pour mesurer l'accuracy et l'cart type (pour valuer la robustesse) mesures sur les donnes utilises pour apprendre le modle (donnes d'apprentissage) et sur des donnes qui n'ont pas t utilises pour mettre au point le modle (donnes test)

La premire approche est gnralement utilise pour comparer plusieurs modles en regardant leur accuracy sur les mmes donnes de test. La deuxime approche est, elle, utilise pour aussi obtenir une certaine valuation de la robustesse du modle appris et garantir que l'accuracy mesure n'est pas simplement obtenue sur "un cas particulier" de donnes.

Cependant, il existe des cas o le dataset que l'on possde ne contient pas beaucoup d'exemples. Dans ce cas, deux possibilits:

  • il y a suffisament de donnes dans le dataset pour raliser une validation croise en rduisant le nombre de paquets (ie. prendre 4 paquets au lieu de 10 par exemple);
  • il y a trop peu d'exemples pour qu'une validation croise soit ralisable, on utilise alors une autre technique d'valuation: l'valuation par leave one out (littralement: "en mettre un de ct").
L'valuation par leave one out procde ainsi: partir d'un dataset $DS$ contenant $n$ exemples:

  1. sortir un des exemples de $DS$ et le mettre de ct

  2. apprendre le modle sur les $n-1$ exemples non sortis de $DS$

  3. tester le modle appris sur l'exemple mis de ct: il est soit bien class (on marque 1 pt), soit mal class (on marque 0 pt)

  4. prendre le $DS$ original est choisir un exemple diffrent mettre de ct

  5. reprendre en 1)

Les tapes 4) et 5) sont faites jusqu' ce que tous les exemples de $DS$ aient t sortis leur tour de $DS$. A la fin, l'accuracy de test du modle est alors donne par le ratio $\frac{\mbox{nombre de points marqus}}{n}$.

Remarque: le plus simple dans cette approche, c'est de sortir les exemples en les prenant dans l'ordre dans lequel ils sont dans le dataset.
